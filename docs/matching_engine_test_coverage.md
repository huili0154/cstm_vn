## 3. 未覆盖场景（风险区域）

### 3.1 高风险 — 与实际策略运行直接相关

| 场景 | 风险等级 | 说明 |
|------|:---:|------|
| **同时双向挂单（买A卖B并发）** | 🔴 高 | 这是 MSTR/DS_DMTR 的核心操作模式，但测试中从未出现同时对两个品种挂单的场景 |
| **被动挂单价格不在盘口 → 长时间不成交** | 🔴 高 | 策略中被动挂单可能挂在非最优价，测试只验证了能成交的情况 |
| **短时间内密集下单/撤单** | 🔴 高 | 策略的追价逻辑会在短时间内反复撤单重挂，未测试状态一致性 |
| **多笔部分成交的资金累积精度** | 🟡 中 | 只有简单的一致性检查（偏差<5万），未验证逐笔资金扣减精确性 |

### 3.2 中等风险 — 边界条件

| 场景 | 风险等级 | 说明 |
|------|:---:|------|
| **涨停/跌停盘口** | 🟡 中 | 单边盘口（ask=0 或 bid=0）下的撮合行为未测试 |
| **盘口为零（集合竞价期间）** | 🟡 中 | 9:15-9:25 / 14:57-15:00 盘口特殊，未测试 |
| **同一品种同方向多笔挂单** | 🟡 中 | 策略的 sub_lots 切片会产生多笔同方向订单，未专门测试优先级 |
| **T+1 在 SMART 模式下** | 🟡 中 | T+1 只在 TICK_FILL 下验证过 |
| **跨日持仓的 available 更新** | 🟡 中 | 隔日后 frozen 解冻逻辑未独立测试 |

### 3.3 低风险 — 极端场景

| 场景 | 风险等级 | 说明 |
|------|:---:|------|
| 价格为 0 的 tick | 🟢 低 | 盘前/盘后可能出现 |
| 超大订单（超过全天成交量） | 🟢 低 | 实际策略不太会触发 |
| 空数据日（停牌） | 🟢 低 | 回测引擎层面已处理 |

## 4. DS_DMTR 策略测试缺失

目前 **ds_dmtr_strategy.py 完全没有测试**，包括：
- 无烟雾测试（策略能否正常运行不崩溃）

- 无 warmup 验证
- 无 block 生命周期测试

## 5. 如何运行测试

```bash
# 运行全部测试
python -m pytest tests/ -v

# 只运行撮合引擎单元测试（不需要数据文件）
python -m pytest tests/test_matching.py -v

# 运行需要真实数据的测试
python -m pytest tests/test_backtest_engine.py tests/test_integration.py -v

# 运行大单卖出观测（手动查看输出）
python tests/test_smart_sell_observation.py
```

## 6. 建议的补充测试优先级

1. **双品种并发挂单测试** — 填补最大风险盲区
2. **密集撤单重挂测试** — 验证追价场景的状态一致性
3. **涨停/跌停/零盘口测试** — 覆盖边界条件
4. **DS_DMTR 策略烟雾测试** — 建立第二个策略的基本保障
5. **资金精度测试** — 多笔部分成交后验证资金逐笔一致
